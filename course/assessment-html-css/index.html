Assessment: Practical Exam on HTML and CSS

Testing Your HTML and CSS Skills

You’ve completed Lessons 10–17 of CODEXEDOC’s Full-Stack Development Course, covering semantic HTML, CSS basics, box model, layouts, responsive design, advanced CSS techniques, SCSS, accessibility, and CSS frameworks. This practical exam tests your ability to apply these skills by building a responsive, accessible e-commerce product card, a common component in web development. By completing this assessment, you’ll demonstrate your readiness to create professional web interfaces.

Why This Assessment Matters

Building user-friendly, responsive, and accessible web components is a core skill for front-end developers. This exam simulates a real-world task—designing a product card for an e-commerce site—testing your ability to combine semantic HTML, CSS styling, and accessibility best practices. Mastering these skills prepares you for creating portfolio projects and contributing to production websites.

Exam Overview

In this practical exam, you’ll create a single e-commerce product card using HTML and CSS (with SCSS or Tailwind CSS). The card should include a header, image placeholder, description, price, and call-to-action button. Requirements include:

  • Semantic HTML: Use tags like <article>, <header>, and <img alt=""> for structure and accessibility.
  • CSS Styling: Apply the box model (padding, margin, border) and style with SCSS (variables, nesting) or Tailwind CSS utility classes.
  • Layout: Use Flexbox or CSS Grid to arrange elements.
  • Responsive Design: Use media queries to adapt the card for mobile (stacked layout) and desktop (fixed width).
  • Advanced CSS: Add a hover animation (e.g., scale or shadow) using transition.
  • Accessibility: Include ARIA attributes, keyboard-navigable buttons, and sufficient color contrast.

Sample Task: Building the Product Card

Here’s an example of the task and a solution:

Task: Create a product card for a “Wireless Headphones” item. Include a header with the product name, an image placeholder, a short description, a price, and an “Add to Cart” button. Style it with SCSS, make it responsive, and ensure accessibility.

Sample Solution (SCSS):

<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Product Card</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <article class="product-card" aria-labelledby="product-title">
        <header>
            <h2 id="product-title">Wireless Headphones</h2>
        </header>
        <img src="placeholder.jpg" alt="Black wireless headphones">
        <p>High-quality headphones with noise cancellation.</p>
        <p class="price">$99.99</p>
        <button aria-label="Add Wireless Headphones to Cart">Add to Cart</button>
    </article>
</body>
</html>

<!-- styles.scss -->
$primary-color: #1e3a8a;
$font-stack: 'Roboto Mono', monospace;

.product-card {
    font-family: $font-stack;
    padding: 20px;
    border: 1px solid #ccc;
    margin: 10px;
    display: flex;
    flex-direction: column;
    gap: 10px;
    transition: transform 0.3s ease;
    &:hover {
        transform: scale(1.05);
    }
    header h2 {
        color: $primary-color;
    }
    img {
        width: 100%;
        max-width: 200px;
    }
    .price {
        font-weight: bold;
    }
    button {
        background: $primary-color;
        color: white;
        padding: 10px;
        border: none;
        cursor: pointer;
        &:hover, &:focus {
            outline: 2px solid yellow;
        }
    }
}

@media (min-width: 768px) {
    .product-card {
        max-width: 300px;
    }
}

Compile styles.scss to styles.css. This solution uses semantic HTML, SCSS variables and nesting, Flexbox, a hover animation, media queries, and accessibility features like ARIA and focus styles.

Tailwind CSS Alternative:

<body class="bg-gray-100">

This Tailwind version uses utility classes to achieve the same result, assuming Tailwind CSS is included.

How to Prepare

To succeed in the exam:

  • Review Lessons: Revisit Lessons 10–17, focusing on semantic tags, box model, Flexbox/Grid, media queries, SCSS, accessibility, and Tailwind CSS.
  • Practice SCSS: Write and compile SCSS with variables and nesting; use sass --watch styles.scss:styles.css.
  • Experiment with Tailwind: Build a simple page with Tailwind classes to learn utilities like flex and md:.
  • Test Accessibility: Use tools like WAVE or a screen reader (NVDA) to check your HTML.
  • Simulate Responsiveness: Use browser dev tools to test layouts at different screen sizes (e.g., 320px, 768px).

Taking the Exam

The exam is hosted on CODEXEDOC’s platform, where you’ll submit your index.html and styles.scss (or CSS file). You have 90 minutes to complete the task. Your submission will be evaluated on:

  • Correctness: Semantic HTML, valid CSS, and functional responsiveness.
  • Accessibility: Proper use of ARIA, alt text, and keyboard navigation.
  • Styling: Consistent use of box model, layouts, and animations.
  • Code Quality: Clean, organized code with minimal errors.

A passing score (80% or higher) earns you an HTML & CSS proficiency badge for your portfolio. If you don’t pass, review feedback and retake after 24 hours.

Summary

This practical exam tests your ability to build a responsive, accessible e-commerce product card, applying HTML and CSS skills from semantic structure to advanced styling. By completing it, you’re ready to tackle the e-commerce product page project and move on to JavaScript. Keep practicing layouts and accessibility, and build with confidence!
