Resolving Merge Conflicts

Welcome to Lesson 8 of CODEXEDOC’s Full-Stack Development Course! As you collaborate on GitHub and use branching strategies, you’ll inevitably encounter merge conflicts—situations where Git can’t automatically combine changes from different branches. This lesson teaches you how to identify, resolve, and prevent merge conflicts, ensuring smooth collaboration.

What is a Merge Conflict?

A merge conflict occurs when two branches modify the same part of a file, and Git can’t decide which changes to keep. For example, if two developers edit the same line in index.html, Git needs your help to resolve the conflict. In 2025, resolving conflicts is a key skill for collaborative development.

Key Concepts

Understand these terms to handle conflicts:

  • Conflict Markers: Git marks conflicts with <<<<<<<, =======, and >>>>>>> in the affected file.
  • Resolve: Manually editing the file to choose or combine conflicting changes.
  • Mark as Resolved: Staging the resolved file with git add and completing the merge with git commit.

Step-by-Step Guide to Resolving Conflicts

Follow these steps to resolve a merge conflict (learn more at 

<a href="https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/addressing-merge-conflicts">docs.github.com</a>
):

  1. Attempt a Merge: Run git merge feature/other from the main branch. If a conflict occurs, Git will pause and notify you.
  2. Locate Conflicts: Open the affected file in your editor (e.g., VS Code) to see conflict markers.
  3. Resolve Conflicts: Edit the file to keep the desired changes, removing conflict markers.
  4. Mark as Resolved: Stage the file with git add <file>.
  5. Complete the Merge: Run git commit to finalize the merge.

Code Example: Resolving a Merge Conflict

Suppose two branches modify index.html:

# On main branch
git checkout main
echo "<h1>Welcome</h1>" > index.html
git add index.html
git commit -m "Update heading on main"

# On feature branch
git checkout -b feature/update
echo "<h1>Hello World</h1>" > index.html
git add index.html
git commit -m "Update heading on feature"

# Merge feature into main
git checkout main
git merge feature/update

# Conflict occurs, edit index.html
<<<<<<< HEAD
<h1>Welcome</h1>
=======
<h1>Hello World</h1>
>>>>>>> feature/update

# Resolve by choosing or combining
<h1>Welcome to My Site</h1>

# Stage and commit
git add index.html
git commit

After resolving, the file contains the combined heading, and the merge is completed.

Practical Applications

Merge conflicts are common in team projects. For example, when two developers edit the same CSS file for a web app, you’ll resolve conflicts to ensure the styles align. In this course, you’ll practice resolving conflicts in the to-do list app project, preparing you for real-world collaboration.

Tips for Success

To handle merge conflicts effectively:

  • Use Visual Tools: Use VS Code’s merge conflict editor or tools like Meld for easier resolution.
  • Communicate with Team: Coordinate with teammates to avoid editing the same files simultaneously.
  • Commit Often: Smaller, frequent commits reduce the scope of conflicts.
  • Back Up: Save a copy of the file before resolving conflicts, just in case.

Summary

Resolving merge conflicts is a critical skill for collaborative development. By understanding conflict markers and following a clear resolution process, you can keep your project on track. In the next lesson, you’ll learn how to automate workflows with GitHub Actions, streamlining your development process. Practice resolving conflicts to build confidence!
