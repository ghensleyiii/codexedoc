Creating RESTful APIs

Welcome to Lesson 30 of CODEXEDOC’s Full-Stack Development Course! RESTful APIs enable communication between frontend and backend using standard HTTP methods. In this lesson, you’ll learn to create a RESTful API with Express.js to manage products for your e-commerce project.

What is a RESTful API?

REST (Representational State Transfer) is an architectural style for designing APIs. RESTful APIs use HTTP methods (GET, POST, PUT, DELETE) to perform CRUD operations (Create, Read, Update, Delete) on resources, like products or users.

Why Build RESTful APIs?

RESTful APIs provide a standardized way for your frontend to interact with the backend, enabling dynamic data updates. They’re widely used in 2025 for scalable web applications, including e-commerce platforms.

Key RESTful API Concepts

Focus on these techniques:

  • HTTP Methods: Use GET (read), POST (create), PUT (update), DELETE (delete).
  • Routes: Define endpoints like /products or /products/:id.
  • Status Codes: Return codes like 200 (OK), 404 (Not Found), or 201 (Created).
  • Request/Response: Handle JSON data with req.body and res.json.

Creating a RESTful API

Follow these steps to build an API:

  1. Set Up Express: Ensure Express is installed and configured.
  2. Define Routes: Create endpoints for CRUD operations.
  3. Handle Requests: Process input and return JSON responses.
  4. Test API: Use Postman to verify endpoints.

Code Example: RESTful Product API

Here’s an Express API for managing products:

// server.js
const express = require("express");
const app = express();

app.use(express.json());

let products = [
    { id: 1, name: "Laptop", price: 999.99 },
    { id: 2, name: "Phone", price: 499.99 }
];

// Get all products
app.get("/api/products", (req, res) => {
    res.json(products);
});

// Get product by ID
app.get("/api/products/:id", (req, res) => {
    const product = products.find(p => p.id === parseInt(req.params.id));
    if (!product) return res.status(404).json({ message: "Product not found" });
    res.json(product);
});

// Create product
app.post("/api/products", (req, res) => {
    const product = {
        id: products.length + 1,
        name: req.body.name,
        price: req.body.price
    };
    products.push(product);
    res.status(201).json(product);
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Test with Postman: GET http://localhost:3000/api/products or POST to /api/products with JSON like { "name": "Tablet", "price": 299.99 }.

Practical Applications

RESTful APIs power e-commerce backends, serving product data or processing orders. In this course, you’ll use this API to fetch products for your frontend cart, enabling real-time updates.

Tips for Success

To master RESTful APIs:

  • Use Clear Endpoints: Follow REST conventions (e.g., /api/products).
  • Validate Input: Check req.body to prevent errors.
  • Return Status Codes: Use appropriate codes for success and errors.
  • Test Thoroughly: Verify all endpoints with Postman.

Summary

RESTful APIs enable seamless frontend-backend communication. By building a product API, you’re ready to connect your e-commerce frontend. In the next lesson, you’ll integrate your frontend with this backend. Keep practicing API development!
