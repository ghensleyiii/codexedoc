Assessment: Practical Exam on Branching and Conflict Resolution

Testing Your Git Branching and Conflict Resolution Skills

You’ve learned the fundamentals of Git, collaboration on GitHub, branching strategies, and conflict resolution in CODEXEDOC’s Git section. This practical exam tests your ability to apply these skills in realistic scenarios. By completing this assessment, you’ll demonstrate your readiness to manage code in professional projects, a critical skill for any developer.

Why This Assessment Matters

In real-world development, branching and conflict resolution are daily tasks. Developers create branches to work on features, bug fixes, or experiments, and they resolve conflicts when multiple changes overlap. This exam simulates these scenarios, ensuring you can confidently use Git in collaborative environments. A strong grasp of these skills will prepare you for team projects and open-source contributions.

Exam Overview

This practical exam involves completing tasks in a Git repository. You’ll be provided with a sample repository containing a simple web project. Your tasks include creating branches, making changes, merging them, and resolving conflicts. The exam includes:

  • Creating and switching branches with git branch and git checkout.
  • Committing changes with clear messages.
  • Merging branches using git merge.
  • Resolving a merge conflict in a provided file.
  • Pushing changes to a GitHub repository and creating a pull request.

Sample Task: Resolving a Merge Conflict

Here’s an example task you might encounter:

Task: You’re working on a repository with a file index.html. Create a branch called feature-title, change the <h1> text to “My App”, and commit. Meanwhile, another developer changes the same line to “Welcome Page” on the main branch. Merge your branch and resolve the conflict.

Sample Conflict:

<<<<<<< HEAD
    <h1>Welcome Page</h1>
=======
    <h1>My App</h1>
>>>>>>> feature-title

Solution: Edit the file to combine changes (e.g., <h1>Welcome to My App</h1>), then run:

git add index.html
git commit -m "Resolve merge conflict in index.html"

This resolves the conflict and completes the merge.

How to Prepare

To succeed in the exam:

  • Review Branching: Practice creating and switching branches with git branch feature-name and git checkout feature-name.
  • Simulate Conflicts: Create a test repository, make conflicting changes on two branches, and practice resolving them.
  • Understand Pull Requests: Review how to create and review pull requests on GitHub.
  • Use Git Commands: Be comfortable with git add, git commit, git merge, and git push.

Taking the Exam

The exam is hosted on CODEXEDOC’s platform, where you’ll access a private repository. You’ll have 60 minutes to complete the tasks. A passing score (80% or higher) earns you a Git proficiency badge for your portfolio. If you don’t pass, review the feedback and retake the exam after 24 hours.

Summary

This practical exam tests your ability to use Git for branching and conflict resolution, key skills for collaborative development. By completing it, you’re ready to move on to HTML and CSS, where you’ll start building web interfaces. Keep practicing Git commands, and you’ll be prepared for real-world projects!
