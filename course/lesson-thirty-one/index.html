Connecting Frontend to Backend

Welcome to Lesson 31 of CODEXEDOC’s Full-Stack Development Course! Connecting your frontend to a backend API enables dynamic data retrieval and updates. In this lesson, you’ll learn to integrate your e-commerce cart (from Lesson 27) with the RESTful API (from Lesson 30) using JavaScript’s Fetch API, ensuring seamless data flow.

What is Frontend-Backend Integration?

Integration involves the frontend sending HTTP requests to the backend API to fetch or modify data, like retrieving products or adding cart items. This creates a seamless user experience in full-stack applications.

Why Connect Frontend to Backend?

Connecting the frontend to a backend allows your app to use real data, persist user actions, and scale functionality. In 2025, this is essential for dynamic e-commerce sites, enabling features like live product updates.

Key Integration Concepts

Focus on these techniques:

  • Fetch API: Use fetch to make HTTP requests from the frontend.
  • Async/Await: Handle API responses asynchronously.
  • Error Handling: Manage API errors with try/catch.
  • CORS: Ensure the backend allows frontend requests with CORS headers.

Connecting the Frontend

Follow these steps to integrate:

  1. Run the Backend: Start your Express server (e.g., node server.js) from Lesson 30.
  2. Serve the Frontend: Use a local server (e.g., npx http-server) to serve index.html.
  3. Update Frontend: Modify the cart to fetch products from the API.
  4. Test Integration: Verify data flows between frontend and backend.

Code Example: Frontend-Backend Integration

Below is the frontend code for the e-commerce cart, fetching products from the API. Ensure the backend (shown next) is running with CORS enabled. Serve index.html via a local server to avoid fetch errors.

<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>E-commerce Cart</title>
    <link href="https://cdn.jsdelivr.net/npm/tailwindcss@2.2.19/dist/tailwind.min.css" rel="stylesheet">
</head>
<body class="bg-gray-100">
    <header class="bg-blue-600 text-white p-4">
        <h1 class="text-2xl font-bold">Store Cart</h1>
    </header>
    <main class="container mx-auto p-4">
        <h2 class="text-xl font-bold mb-4">Products</h2>
        <div id="product-list" class="grid grid-cols-1 md:grid-cols-2 gap-4"></div>
        <h2 class="text-xl font-bold mt-8 mb-4">Cart</h2>
        <ul id="cart-list" class="bg-white p-4 rounded shadow"></ul>
        <p id="total" class="mt-4 font-semibold">Total: $0.00</p>
    </main>
    <script>
        const cart = [];
        const productList = document.querySelector("#product-list");
        const cartList = document.querySelector("#cart-list");
        const totalDisplay = document.querySelector("#total");

        async function fetchProducts() {
            try {
                const response = await fetch("http://localhost:3000/api/products", {
                    method: "GET",
                    headers: { "Content-Type": "application/json" }
                });
                if (!response.ok) {
                    throw new Error(`HTTP error! Status: ${response.status}`);
                }
                const products = await response.json();
                renderProducts(products);
            } catch (error) {
                console.error("Fetch error:", error.message);
                productList.innerHTML = "<p class='text-red-600'>Error loading products. Ensure the backend server is running at http://localhost:3000 and CORS is enabled.</p>";
            }
        }

        function renderProducts(products) {
            productList.innerHTML = "";
            products.forEach(product => {
                const div = document.createElement("div");
                div.className = "bg-white p-4 rounded shadow";
                div.innerHTML = `
                    <h3 class="font-bold">${product.name}</h3>
                    <p>$${product.price.toFixed(2)}</p>
                    <button class="bg-blue-600 text-white px-4 py-2 rounded" onclick="addToCart(${product.id}, '${product.name}', ${product.price})">Add to Cart</button>
                `;
                productList.appendChild(div);
            });
        }

        function addToCart(id, name, price) {
            const item = cart.find(item => item.id === id);
            if (item) {
                item.quantity++;
            } else {
                cart.push({ id, name, price: parseFloat(price), quantity: 1 });
            }
            renderCart();
        }

        function renderCart() {
            cartList.innerHTML = "";
            let total = 0;
            cart.forEach(item => {
                total += item.price * item.quantity;
                const li = document.createElement("li");
                li.className = "flex justify-between py-2";
                li.innerHTML = `
                    <span>${item.name} (x${item.quantity}) - $${(item.price * item.quantity).toFixed(2)}</span>
                    <button class="text-red-600" onclick="removeFromCart(${item.id})">Remove</button>
                `;
                cartList.appendChild(li);
            });
            totalDisplay.textContent = `Total: $${total.toFixed(2)}`;
        }

        function removeFromCart(id) {
            const index = cart.findIndex(item => item.id === id);
            if (index !== -1) {
                cart.splice(index, 1);
                renderCart();
            }
        }

        fetchProducts();
    </script>
</body>
</html>

Backend Requirement: The frontend requires the Lesson 30 backend with CORS enabled. Save the following as server.js, install dependencies (npm install express cors), and run with node server.js:

// server.js
const express = require("express");
const cors = require("cors");
const app = express();

app.use(cors());
app.use(express.json());

let products = [
    { id: 1, name: "Laptop", price: 999.99 },
    { id: 2, name: "Phone", price: 499.99 }
];

app.get("/api/products", (req, res) => {
    res.json(products);
});

app.get("/api/products/:id", (req, res) => {
    const product = products.find(p => p.id === parseInt(req.params.id));
    if (!product) return res.status(404).json({ message: "Product not found" });
    res.json(product);
});

app.post("/api/products", (req, res) => {
    const product = {
        id: products.length + 1,
        name: req.body.name,
        price: req.body.price
    };
    products.push(product);
    res.status(201).json(product);
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Setup Instructions:

  1. Save the frontend code as index.html.
  2. Save the backend code as server.js, install dependencies, and run node server.js.
  3. Serve index.html using a local server (e.g., npx http-server or VS Code Live Server) to avoid CORS issues with file://.
  4. Test the backend by visiting http://localhost:3000/api/products in a browser (should return JSON).
  5. Open the frontend in a browser via the local server (e.g., http://localhost:8080).

Troubleshooting "Failed to fetch": If the error persists:

  • Check Backend: Ensure http://localhost:3000/api/products returns JSON in a browser.
  • Inspect CORS: Open DevTools (F12) > Console for CORS errors. The cors middleware should resolve this.
  • Verify Port: Confirm no other service uses port 3000 (e.g., run lsof -i :3000 on Unix).
  • Network Issues: Disable firewalls or test on a different network.
  • Debug Request: In DevTools > Network, check the fetch request’s status (e.g., 404, 500, or "Failed").
  • File Protocol: If opening index.html directly (file://), use a local server instead.

Practical Applications

Frontend-backend integration powers real-time e-commerce features, like updating product availability. In this course, this integration makes your cart fully dynamic, preparing you for full-stack development.

Tips for Success

To master integration:

  • Enable CORS: Use app.use(cors()) in Express to allow frontend requests.
  • Serve Locally: Always use a local server for the frontend to avoid fetch issues.
  • Handle Errors: Display user-friendly error messages, as shown in the code.
  • Debug Requests: Use DevTools’ Network tab to inspect API calls and status codes.

Summary

Connecting your frontend to a backend creates dynamic, data-driven apps. By integrating your cart with an API, you’ve built a full-stack feature. In the next lesson, you’ll learn about databases to persist data. Keep practicing integration!
