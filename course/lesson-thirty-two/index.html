Introduction to Databases with MongoDB

Welcome to Lesson 32 of CODEXEDOC’s Full-Stack Development Course! Databases store and manage data for web applications, like products or user profiles. In this lesson, you’ll learn about MongoDB, a NoSQL database, and set it up to persist data for your e-commerce API.

What is MongoDB?

MongoDB is a NoSQL database that stores data in flexible, JSON-like documents. Unlike SQL databases, it uses collections and documents, making it ideal for scalable, modern apps in 2025, especially with JavaScript-based backends.

Why Use MongoDB?

MongoDB’s document model aligns with JavaScript objects, simplifying integration with Node.js and Express. It’s perfect for e-commerce apps needing to store dynamic product data or user information.

Key MongoDB Concepts

Focus on these techniques:

  • Collections: Groups of documents, similar to tables in SQL.
  • Documents: JSON-like records, e.g., { "name": "Laptop", "price": 999.99 }.
  • Mongoose: An ODM (Object Data Modeling) library for MongoDB and Node.js.
  • Connection: Connect to MongoDB using a URI (local or cloud).

Setting Up MongoDB

Follow these steps to use MongoDB:

  1. Install MongoDB: Use MongoDB Community or MongoDB Atlas (cloud).
  2. Install Mongoose: Run npm install mongoose.
  3. Connect to MongoDB: Configure a connection in your Express app.
  4. Test Connection: Verify with a simple document insertion.

Code Example: MongoDB Setup

Here’s an Express app connected to MongoDB:

// server.js
const express = require("express");
const mongoose = require("mongoose");
const app = express();

app.use(express.json());

mongoose.connect("mongodb://localhost:27017/ecommerce", {
    useNewUrlParser: true,
    useUnifiedTopology: true
}).then(() => {
    console.log("Connected to MongoDB");
}).catch(err => {
    console.error("MongoDB connection error:", err);
});

const productSchema = new mongoose.Schema({
    name: String,
    price: Number
});

const Product = mongoose.model("Product", productSchema);

app.get("/api/products", async (req, res) => {
    try {
        const products = await Product.find();
        res.json(products);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Run MongoDB locally, then node server.js. This connects to a local MongoDB instance and fetches products.

Practical Applications

MongoDB stores product catalogs, user data, or order histories in e-commerce apps. In this course, you’ll use MongoDB to persist products, making your API fully functional.

Tips for Success

To master MongoDB:

  • Use Atlas for Simplicity: Try MongoDB Atlas for cloud hosting.
  • Validate Schemas: Define strict schemas with Mongoose.
  • Handle Errors: Catch connection and query errors.
  • Test Locally: Use MongoDB Compass to inspect data.

Summary

MongoDB provides a flexible, scalable database for your backend. By setting up MongoDB, you’re ready to store data persistently. In the next lesson, you’ll learn CRUD operations to manage data. Keep exploring MongoDB!
