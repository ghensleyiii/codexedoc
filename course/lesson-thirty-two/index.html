Testing APIs (Supertest)

Mastering API Testing with Supertest

Welcome to Lesson 32! Testing APIs ensures reliable endpoints. In this lesson, you’ll learn to use Supertest, a library for testing Express APIs, to verify responses and behavior.

What is Supertest?

Supertest is a Node.js library for HTTP assertions, built for Express. In 2025, it’s a standard for testing REST APIs, often paired with Jest for comprehensive coverage.

Key Concepts

Focus on these techniques:

  • HTTP Assertions: Test status codes and response bodies.
  • Integration Tests: Simulate requests to endpoints.
  • Mocks: Mock databases or services.
  • Test Setup: Initialize Express app for testing.

Code Example: Supertest

Here’s a test for an Express API:


const express = require('express');
const app = express();

app.use(express.json());

app.get('/api/posts', (req, res) => {
    res.json([{ id: 1, title: 'Post 1' }]);
});

module.exports = app;


const request = require('supertest');
const app = require('./server');

describe('API Endpoints', () => {
    it('should get posts', async () => {
        const res = await request(app)
            .get('/api/posts')
            .expect(200);
        expect(res.body).toEqual([{ id: 1, title: 'Post 1' }]);
    });
});

Run jest to test the /api/posts endpoint.

Practical Applications

Supertest ensures API reliability. In your blog API project, you’ll test endpoints for creating and retrieving posts.

Tips for Success

To master Supertest:

  • Use Jest: Combine with Supertest for assertions.
  • Test Status Codes: Verify 200, 404, etc.
  • Mock Dependencies: Use libraries like sinon.
  • Run Tests: Use npm test for continuous testing.

Summary

Supertest ensures robust APIs through testing. By mastering it, you’re ready for production backends. Next, you’ll learn to deploy Node.js apps. Keep testing your APIs!
