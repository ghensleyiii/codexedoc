Building a Professional Portfolio

Welcome to Lesson 25! You’ll learn to build a portfolio showcasing your full-stack skills. By the end, you’ll host a portfolio page online.

Introduction

A portfolio is like a shepherd displaying prized wool, highlighting your work to employers. This lesson covers selecting projects and hosting on GitHub Pages or Netlify.

Key Concepts

Focus areas include:

  • Project Selection: Choose exam outputs (webpage, API, SQL).
  • Structure: Include homepage, about, projects, contact.
  • Hosting: Use GitHub Pages or Netlify for free deployment.

Code Example

Here’s a simple portfolio homepage with HTML and CSS.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Shepherd's Portfolio</title>
    <style>
        body { font-family: Arial, sans-serif; text-align: center; }
        .project { margin: 20px; }
    </style>
</head>
<body>
    <header>
        <h1>Shepherd's Portfolio</h1>
        <nav>
            <a href="#about">About</a> | <a href="#projects">Projects</a> | <a href="#contact">Contact</a>
        </nav>
    </header>
    <section id="projects">
        <h2>Projects</h2>
        <div class="project">
            <h3>To-Do API</h3>
            <p>Express API with JWT authentication.</p>
        </div>
    </section>
</body>
</html>

Save as index.html, push to a GitHub repository, and enable GitHub Pages in the repository settings.

Practical Applications

Portfolios secure jobs at companies like Google by showcasing live projects. They demonstrate skills like API development or responsive design, critical for full-stack roles.

Tips for Success

Build a strong portfolio with these tips:

  • Include 3–5 polished projects with source code links.
  • Write clear, concise project descriptions.
  • Ensure mobile-friendly, accessible design.

Shearing Tip

A portfolio shears away anonymity by showcasing your skills, like a shepherd presenting wool to buyers.
