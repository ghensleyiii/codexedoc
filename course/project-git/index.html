Project: Collaborate on a Team-Based GitHub To-Do List App

Collaborating on a To-Do List App with GitHub

In this project, you’ll apply your Git skills to collaborate on a team-based GitHub repository, building a simple to-do list application. This hands-on exercise simulates real-world development, where multiple developers work together using version control. You’ll use Git commands, GitHub workflows, and HTML/CSS to create a functional app, reinforcing the concepts you’ve learned in the Git section.

Project Overview

Your goal is to contribute to a shared GitHub repository to build a to-do list app. The app will allow users to add, view, and delete tasks, styled with basic HTML and CSS. You’ll work in a team (or simulate one using multiple branches), using Git branches, commits, and pull requests to manage contributions. This project introduces you to collaborative workflows used in professional development teams.

Step-by-Step Guide

Follow these steps to complete the project:

  1. Set Up the Repository: Fork the project repository on GitHub and clone it to your local machine using git clone. Create a new branch for your feature (e.g., git branch add-task).
  2. Build the HTML Structure: Create an index.html file with a form to add tasks and a list to display them. Use semantic tags like <ul> for the task list.
  3. Style with CSS: Add styles to make the app user-friendly. Use CSS to align the form and list, and add hover effects for interactivity.
  4. Commit and Push Changes: Stage your changes with git add ., commit them with git commit -m "Add task form and styles", and push to your branch with git push origin add-task.
  5. Create a Pull Request: On GitHub, create a pull request to merge your branch into the main repository. Review another teammate’s pull request to practice collaboration.
  6. Test the App: Open the app in a browser to ensure the form and list display correctly. Check for responsiveness on mobile devices.

Code Example: To-Do List Structure

Here’s a sample HTML and CSS for the to-do list app:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>To-Do List</title>
    <style>
        body { font-family: Roboto, sans-serif; max-width: 600px; margin: auto; padding: 20px; }
        input { padding: 10px; width: 80%; }
        button { padding: 10px; }
        ul { list-style: none; padding: 0; }
        li { padding: 10px; background: #f9f9f9; margin: 5px 0; }
    </style>
</head>
<body>
    <h1>To-Do List</h1>
    <form>
        <input type="text" placeholder="Add a task">
        <button type="submit">Add</button>
    </form>
    <ul>
        <li>Learn Git</li>
        <li>Build project</li>
    </ul>
</body>
</html>

This code creates a simple to-do list interface. Later in the course, you’ll add JavaScript to make it interactive. For now, focus on using Git to manage your contributions.

Tips for Collaboration

To succeed in this collaborative project:

  • Use Descriptive Commits: Write clear commit messages (e.g., “Add task input form”) to help teammates understand your changes.
  • Communicate: If working with others, discuss branch names and responsibilities to avoid conflicts.
  • Review Pull Requests: Provide constructive feedback on teammates’ code to practice code review skills.
  • Test Before Pushing: Ensure your changes work locally before pushing to GitHub.

Why This Project Matters

Collaboration is a core skill in software development. Most projects involve multiple developers using Git and GitHub to manage code. This project prepares you for team workflows by teaching you to create branches, commit changes, and merge via pull requests. The to-do list app is a practical starting point, and you’ll enhance it with JavaScript later in the course.

Summary

In this project, you’ve used Git and GitHub to collaborate on a to-do list app, practicing branching, committing, and pull requests. These skills are essential for working in development teams. Save your repository link for your portfolio, and prepare for the Git assessment to test your version control knowledge!
