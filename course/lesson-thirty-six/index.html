Securing API Endpoints

Welcome to Lesson 36 of CODEXEDOC’s Full-Stack Development Course! Securing API endpoints ensures only authorized users access sensitive data. In this lesson, you’ll learn to protect your e-commerce API routes using JWT middleware, restricting access to authenticated users.

What is API Security?

API security involves protecting endpoints from unauthorized access, using techniques like token verification and input validation. In 2025, securing APIs is critical for safeguarding user data in web applications.

Why Secure Endpoints?

Securing endpoints prevents unauthorized actions, like modifying products without permission. For your e-commerce app, this ensures only logged-in users can perform sensitive operations.

Key Security Concepts

Focus on these techniques:

  • JWT Middleware: Verify tokens before processing requests.
  • Authorization: Restrict routes to specific users.
  • Input Validation: Sanitize request data to prevent attacks.
  • Error Handling: Return clear error messages for unauthorized access.

Securing Endpoints

Follow these steps to secure your API:

  1. Create Middleware: Write a function to verify JWTs.
  2. Protect Routes: Apply middleware to sensitive endpoints.
  3. Validate Input: Check request data integrity.
  4. Test Security: Use Postman to verify protected routes.

Code Example: Securing Endpoints

Here’s an Express API with a protected product creation route (extends Lesson 35):

// server.js
const express = require("express");
const mongoose = require("mongoose");
const jwt = require("jsonwebtoken");
const cors = require("cors");
const app = express();

app.use(cors());
app.use(express.json());

mongoose.connect("mongodb://localhost:27017/ecommerce", {
    useNewUrlParser: true,
    useUnifiedTopology: true
}).then(() => console.log("Connected to MongoDB"));

const productSchema = new mongoose.Schema({
    name: { type: String, required: true },
    price: { type: Number, required: true }
});

const Product = mongoose.model("Product", productSchema);

// JWT Middleware
const authMiddleware = (req, res, next) => {
    const token = req.header("Authorization")?.replace("Bearer ", "");
    if (!token) return res.status(401).json({ message: "No token provided" });
    try {
        const decoded = jwt.verify(token, "secret_key");
        req.user = decoded;
        next();
    } catch (err) {
        res.status(401).json({ message: "Invalid token" });
    }
};

// Protected Route
app.post("/api/products", authMiddleware, async (req, res) => {
    try {
        const product = new Product(req.body);
        await product.save();
        res.status(201).json(product);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

// Public Route
app.get("/api/products", async (req, res) => {
    try {
        const products = await Product.find();
        res.json(products);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Test with Postman: POST /api/products with a Bearer token from Lesson 35’s login. Without a valid token, you’ll get a 401 error.

Practical Applications

Securing endpoints protects e-commerce APIs from unauthorized access, ensuring only admins can add products. This enhances your app’s security and reliability.

Tips for Success

To master API security:

  • Use HTTPS: Secure API traffic in production.
  • Validate Tokens: Always verify JWTs before processing.
  • Sanitize Inputs: Use libraries like express-validator.
  • Test Thoroughly: Try accessing routes without tokens.

Summary

Securing API endpoints with JWT middleware protects your e-commerce app. By implementing authentication checks, you’ve enhanced security. In the next lesson, you’ll learn to handle file uploads. Keep practicing security!
