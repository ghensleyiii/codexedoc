CRUD Operations with MongoDB

Welcome to Lesson 33 of CODEXEDOC’s Full-Stack Development Course! CRUD (Create, Read, Update, Delete) operations are the core of database interactions. In this lesson, you’ll learn to implement CRUD operations with MongoDB and Mongoose to manage products in your e-commerce API.

What are CRUD Operations?

CRUD operations allow you to create, read, update, and delete data in a database. In MongoDB, these operations are performed on documents within collections, enabling dynamic data management for web apps.

Why Use CRUD with MongoDB?

CRUD operations make your API fully functional, allowing your e-commerce app to store, retrieve, modify, and remove products persistently. Mongoose simplifies these operations with a JavaScript-friendly interface.

Key CRUD Concepts

Focus on these techniques:

  • Create: Add documents with Model.create or save.
  • Read: Retrieve data with find, findOne, or findById.
  • Update: Modify documents with updateOne or findByIdAndUpdate.
  • Delete: Remove documents with deleteOne or findByIdAndDelete.

Implementing CRUD Operations

Follow these steps to add CRUD to your API:

  1. Define a Schema: Create a Mongoose schema for products.
  2. Create Routes: Add Express routes for each CRUD operation.
  3. Handle Requests: Use Mongoose methods to interact with MongoDB.
  4. Test Operations: Use Postman to verify CRUD functionality.

Code Example: CRUD API with MongoDB

Here’s an Express API with full CRUD for products:

// server.js
const express = require("express");
const mongoose = require("mongoose");
const app = express();

app.use(express.json());

mongoose.connect("mongodb://localhost:27017/ecommerce", {
    useNewUrlParser: true,
    useUnifiedTopology: true
}).then(() => console.log("Connected to MongoDB"));

const productSchema = new mongoose.Schema({
    name: { type: String, required: true },
    price: { type: Number, required: true }
});

const Product = mongoose.model("Product", productSchema);

// Create
app.post("/api/products", async (req, res) => {
    try {
        const product = new Product(req.body);
        await product.save();
        res.status(201).json(product);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

// Read all
app.get("/api/products", async (req, res) => {
    try {
        const products = await Product.find();
        res.json(products);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

// Read one
app.get("/api/products/:id", async (req, res) => {
    try {
        const product = await Product.findById(req.params.id);
        if (!product) return res.status(404).json({ message: "Product not found" });
        res.json(product);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

// Update
app.put("/api/products/:id", async (req, res) => {
    try {
        const product = await Product.findByIdAndUpdate(req.params.id, req.body, {
            new: true,
            runValidators: true
        });
        if (!product) return res.status(404).json({ message: "Product not found" });
        res.json(product);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

// Delete
app.delete("/api/products/:id", async (req, res) => {
    try {
        const product = await Product.findByIdAndDelete(req.params.id);
        if (!product) return res.status(404).json({ message: "Product not found" });
        res.json({ message: "Product deleted" });
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Test with Postman: POST /api/products with { "name": "Tablet", "price": 299.99 }, GET /api/products, etc.

Practical Applications

CRUD operations enable e-commerce apps to manage product inventories dynamically. In this course, you’ll use these operations to power your API, allowing your frontend to interact with persistent data.

Tips for Success

To master CRUD:

  • Validate Data: Use Mongoose validators to ensure data integrity.
  • Handle Errors: Return meaningful error messages.
  • Test All Operations: Verify each CRUD endpoint works.
  • Use Async/Await: Keep code readable with async syntax.

Summary

CRUD operations with MongoDB make your API fully functional, enabling data management. By implementing CRUD, you’ve built a robust backend. In the next lesson, you’ll create an e-commerce API project. Keep practicing CRUD!
