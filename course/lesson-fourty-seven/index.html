CI/CD for Full-Stack Apps

Welcome to Lesson 47 of CODEXEDOC’s Full-Stack Development Course! Continuous Integration and Continuous Deployment (CI/CD) automate testing and deployment. In this lesson, you’ll set up a CI/CD pipeline for your e-commerce app using GitHub Actions, ensuring reliable updates.

What is CI/CD?

CI/CD automates building, testing, and deploying code. In 2025, it’s essential for rapid, error-free releases in full-stack development.

Why Use CI/CD?

CI/CD reduces manual errors and speeds up deployment for your e-commerce app, ensuring consistent updates to frontend and backend.

Key CI/CD Concepts

Focus on these techniques:

  • Continuous Integration: Automatically test code on commits.
  • Continuous Deployment: Deploy passing builds to production.
  • GitHub Actions: Define workflows for automation.
  • Environment Variables: Securely manage secrets.

Setting Up CI/CD

Follow these steps:

  1. Create Workflow: Define a GitHub Actions workflow.
  2. Test Frontend/Backend: Run unit tests.
  3. Deploy to Vercel: Automate deployment for frontend.
  4. Deploy to Render: Deploy backend with MongoDB.

Code Example: CI/CD Pipeline

Here’s a GitHub Actions workflow for your e-commerce app:


name: CI/CD Pipeline
on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install frontend dependencies
        run: npm install
        working-directory: ./frontend
      - name: Run frontend tests
        run: npm test
        working-directory: ./frontend
      - name: Install backend dependencies
        run: npm install
        working-directory: ./backend
      - name: Run backend tests
        run: npm test
        working-directory: ./backend

  deploy-frontend:
    needs: test
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to Vercel
        run: npx vercel --prod --token ${{ secrets.VERCEL_TOKEN }}
        working-directory: ./frontend
        env:
          VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
          VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}

  deploy-backend:
    needs: test
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to Render
        run: curl -X POST ${{ secrets.RENDER_DEPLOY_HOOK }}

const mongoose = require('mongoose');
const Product = require('../models/Product');

describe('Product Model', () => {
  beforeAll(async () => {
    await mongoose.connect('mongodb://localhost:27017/test', {
      useNewUrlParser: true,
      useUnifiedTopology: true
    });
  });

  afterAll(async () => {
    await mongoose.connection.close();
  });

  it('should create a product', async () => {
    const product = new Product({ name: 'Test', price: 99.99 });
    const saved = await product.save();
    expect(saved.name).toBe('Test');
  });
});

Structure: frontend/ for React, backend/ for Express. Install: npm install --save-dev jest @testing-library/react (frontend), npm install --save-dev jest (backend). Store secrets in GitHub Settings. Deploy frontend to Vercel, backend to Render.

Practical Applications

CI/CD automates testing and deployment for your e-commerce app, enabling faster, safer releases and scalability.

Tips for Success

To master CI/CD:

  • Write Tests: Ensure robust test coverage.
  • Secure Secrets: Use GitHub Secrets for tokens.
  • Monitor Builds: Check Action logs for errors.
  • Incremental Deployment: Test locally before pushing.
