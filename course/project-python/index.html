Project: Build a Social Media Dashboard with Django

Building a Social Media Dashboard

You’ve completed Lessons 34–40, mastering Python, Flask, Django, DRF, databases, and Pytest. In this project, you’ll build a social media dashboard with Django, integrating REST APIs, authentication, and a frontend, showcasing your full-stack skills.

Why This Project Matters

Social media dashboards are common in real-world apps, requiring secure APIs and dynamic frontends. This project simulates a professional task, preparing you for full-stack roles and portfolio enhancement.

Project Overview

You’ll create a dashboard allowing users to post, view, and like content, with a REST API (DRF) and a React or Django template frontend. Use PostgreSQL for data storage. Requirements include:

  • REST API: Build endpoints for users, posts, and likes using DRF.
  • Authentication: Implement JWT for secure access.
  • Database: Use PostgreSQL with Django ORM.
  • Frontend: Create a UI with React or Django templates.
  • Testing: Write Pytest tests for API endpoints.
  • Deployment: Deploy to Heroku or similar.

Sample Implementation

Here’s a simplified Django API and frontend:

# models.py
from django.db import models
from django.contrib.auth.models import User

class Post(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

# serializers.py
from rest_framework import serializers
from .models import Post

class PostSerializer(serializers.ModelSerializer):
    class Meta:
        model = Post
        fields = ['id', 'user', 'content', 'created_at']

# views.py
from rest_framework import viewsets
from .models import Post
from .serializers import PostSerializer

class PostViewSet(viewsets.ModelViewSet):
    queryset = Post.objects.all()
    serializer_class = PostSerializer

# urls.py
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from . import views

router = DefaultRouter()
router.register(r'posts', views.PostViewSet)

urlpatterns = [
    path('api/', include(router.urls)),
]

# templates/dashboard.html
<ul>
{% for post in posts %}
    <li>{{ post.content }} - {{ post.user.username }}</li>
{% endfor %}
</ul>

# test_api.py
import pytest
from rest_framework.test import APIClient
from django.contrib.auth.models import User
from .models import Post

@pytest.fixture
def client():
    return APIClient()

@pytest.fixture
def user():
    return User.objects.create_user(username='testuser', password='testpass')

def test_create_post(client, user):
    client.force_authenticate(user=user)
    response = client.post('/api/posts/', {'content': 'Test post'})
    assert response.status_code == 201
    assert response.data['content'] == 'Test post'

How to Approach the Project

Follow these steps:

  • Setup: Initialize a Django project with django-admin startproject.
  • Models: Define User, Post, and Like models.
  • API: Create DRF viewsets and serializers.
  • Authentication: Add JWT with djangorestframework-simplejwt.
  • Frontend: Build with React or Django templates.
  • Testing: Write Pytest tests for API endpoints.
  • Deploy: Host on Heroku with PostgreSQL.

Tips for Success

To excel:

  • Plan Models: Design database schema first.
  • Secure APIs: Use JWT and permissions.
  • Test Thoroughly: Cover API and frontend logic.
  • Document: Include a README with setup instructions.
  • Style Frontend: Use Tailwind CSS or Bootstrap.
