Handling File Uploads

Welcome to Lesson 37 of CODEXEDOC’s Full-Stack Development Course! File uploads allow users to send files, like product images, to your server. In this lesson, you’ll learn to handle file uploads in your e-commerce API using Express and Multer, enhancing product management.

What are File Uploads?

File uploads involve sending files from the client to the server, typically via multipart form data. In 2025, handling uploads securely is essential for features like user avatars or product images.

Why Handle File Uploads?

Uploads enable visual content in e-commerce apps, like product photos. Integrating uploads with your API allows users to add images, improving the shopping experience.

Key Upload Concepts

Focus on these techniques:

  • Multer: A middleware for handling multipart form data.
  • File Storage: Save files to the server or cloud.
  • Validation: Restrict file types and sizes.
  • Integration: Store file metadata in MongoDB.

Setting Up File Uploads

Follow these steps to handle uploads:

  1. Install Multer: Add multer to your project.
  2. Configure Storage: Set up Multer to save files.
  3. Create Endpoint: Add a route for file uploads.
  4. Test Uploads: Use Postman or a frontend form.

Code Example: File Uploads

Here’s an Express API with file uploads (extends Lesson 36):

// server.js
const express = require("express");
const mongoose = require("mongoose");
const multer = require("multer");
const cors = require("cors");
const app = express();

app.use(cors());
app.use(express.json());
app.use("/uploads", express.static("uploads"));

mongoose.connect("mongodb://localhost:27017/ecommerce", {
    useNewUrlParser: true,
    useUnifiedTopology: true
}).then(() => console.log("Connected to MongoDB"));

const productSchema = new mongoose.Schema({
    name: { type: String, required: true },
    price: { type: Number, required: true },
    image: { type: String }
});

const Product = mongoose.model("Product", productSchema);

const storage = multer.diskStorage({
    destination: (req, file, cb) => {
        cb(null, "uploads/");
    },
    filename: (req, file, cb) => {
        cb(null, Date.now() + "-" + file.originalname);
    }
});

const upload = multer({
    storage,
    fileFilter: (req, file, cb) => {
        if (file.mimetype.startsWith("image/")) {
            cb(null, true);
        } else {
            cb(new Error("Invalid file type"), false);
        }
    }
});

app.post("/api/products", upload.single("image"), async (req, res) => {
    try {
        const product = new Product({
            name: req.body.name,
            price: req.body.price,
            image: req.file ? `/uploads/${req.file.filename}` : null
        });
        await product.save();
        res.status(201).json(product);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

app.listen(3000, () => {
    console.log("Server running at http://localhost:3000/");
});

Create an uploads folder. Install dependencies: npm install express mongoose multer cors. Test with Postman: POST /api/products with form-data (key: image, file; keys: name, price, text).

Practical Applications

File uploads enhance e-commerce apps by allowing product images, improving user engagement. In this course, you’ll use uploads to enrich your product catalog.

Tips for Success

To master file uploads:

  • Validate Files: Restrict to images and set size limits.
  • Use Cloud Storage: Consider AWS S3 for production.
  • Secure Uploads: Protect the upload endpoint with JWT.
  • Test Locally: Ensure the uploads folder exists.
