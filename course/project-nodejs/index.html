Project: Build a RESTful Blog API with Authentication

Building a RESTful Blog API

You’ve completed Lessons 27–33, mastering Node.js, Express, authentication, API design, security, testing, and deployment. In this project, you’ll build a RESTful blog API with user authentication, a portfolio-worthy backend showcasing your skills.

Why This Project Matters

Blog APIs power content platforms, requiring secure, scalable endpoints. This project simulates a professional task, preparing you for backend development roles and demonstrating your Node.js expertise.

Project Overview

You’ll create a REST API for a blog with endpoints for users and posts, using Express, JWT authentication, MongoDB (or a mock database), and Supertest for testing. Requirements include:

  • REST Endpoints: Create routes for user registration, login, and CRUD operations for posts.
  • Authentication: Secure post creation with JWT.
  • Database: Store users and posts in MongoDB or a mock store.
  • Security: Use environment variables and input validation.
  • Testing: Write Supertest tests for all endpoints.
  • Deployment: Deploy to a cloud platform like Heroku.

Sample Implementation

Here’s a simplified blog API:


const express = require('express');
const jwt = require('jsonwebtoken');
const dotenv = require('dotenv');
const app = express();

dotenv.config();
app.use(express.json());

const posts = [];
const SECRET_KEY = process.env.SECRET_KEY || 'your-secret-key';

const authMiddleware = (req, res, next) => {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ message: 'No token' });
    try {
        const decoded = jwt.verify(token, SECRET_KEY);
        req.user = decoded;
        next();
    } catch (error) {
        res.status(401).json({ message: 'Invalid token' });
    }
};

app.post('/api/posts', authMiddleware, (req, res) => {
    const { title, content } = req.body;
    const post = { id: posts.length + 1, title, content, author: req.user.username };
    posts.push(post);
    res.status(201).json(post);
});

app.get('/api/posts', (req, res) => {
    res.json(posts);
});

app.listen(3000, () => {
    console.log('Server running on port 3000');
});


const request = require('supertest');
const app = require('./server');

describe('Blog API', () => {
    it('should get all posts', async () => {
        const res = await request(app)
            .get('/api/posts')
            .expect(200);
        expect(Array.isArray(res.body)).toBe(true);
    });
});

This API includes a protected POST /api/posts endpoint and a Supertest test. Add user routes, MongoDB, and deployment for the full project.

How to Approach the Project

Follow these steps:

  • Setup: Initialize with npm init and install Express, JWT, dotenv, and MongoDB.
  • Routes: Create /api/users and /api/posts endpoints.
  • Authentication: Implement JWT for protected routes.
  • Database: Use MongoDB or a mock array for data.
  • Security: Add helmet and input validation.
  • Testing: Write Supertest tests for all endpoints.
  • Deploy: Push to Heroku with environment variables.

Tips for Success

To excel:

  • Plan Endpoints: Define REST routes before coding.
  • Secure Tokens: Store JWT secrets in .env.
  • Test Thoroughly: Cover success and error cases.
  • Document: Include a README with API usage.
  • Monitor Deployment: Check logs on the cloud platform.
