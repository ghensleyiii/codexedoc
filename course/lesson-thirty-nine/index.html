Testing APIs with Jest

Welcome to Lesson 39 of CODEXEDOC’s Full-Stack Development Course! Testing ensures your API works as expected. In this lesson, you’ll learn to test your e-commerce API with Jest and Supertest, verifying endpoints like product creation and retrieval.

What is Jest?

Jest is a JavaScript testing framework for unit and integration tests. Paired with Supertest, it’s ideal for testing Express APIs, ensuring reliability in 2025’s development landscape.

Why Test APIs?

Testing catches bugs early, ensuring your e-commerce API handles requests correctly. Automated tests save time and improve confidence in production code.

Key Testing Concepts

Focus on these techniques:

  • Unit Tests: Test individual functions or routes.
  • Integration Tests: Test API endpoints with Supertest.
  • Mocks: Simulate MongoDB interactions.
  • Assertions: Use Jest’s expect to verify responses.

Setting Up Jest

Follow these steps to test your API:

  1. Install Jest/Supertest: Add testing packages.
  2. Create Test File: Write tests for API endpoints.
  3. Configure MongoDB: Use an in-memory database for tests.
  4. Run Tests: Execute with npm test.

Code Example: Testing API

Here’s a test suite for the Lesson 38 API:

// product.test.js
const request = require("supertest");
const express = require("express");
const mongoose = require("mongoose");
const { MongoMemoryServer } = require("mongodb-memory-server");

const app = express();
app.use(express.json());

const productSchema = new mongoose.Schema({
    name: { type: String, required: true },
    price: { type: Number, required: true }
});

const Product = mongoose.model("Product", productSchema);

app.get("/api/products", async (req, res) => {
    const products = await Product.find();
    res.json(products);
});

app.post("/api/products", async (req, res) => {
    const product = new Product(req.body);
    await product.save();
    res.status(201).json(product);
});

describe("Product API", () => {
    let mongoServer;

    beforeAll(async () => {
        mongoServer = await MongoMemoryServer.create();
        const uri = mongoServer.getUri();
        await mongoose.connect(uri);
    });

    afterAll(async () => {
        await mongoose.disconnect();
        await mongoServer.stop();
    });

    afterEach(async () => {
        await Product.deleteMany({});
    });

    it("should create a product", async () => {
        const res = await request(app)
            .post("/api/products")
            .send({ name: "Laptop", price: 999.99 });
        expect(res.status).toBe(201);
        expect(res.body.name).toBe("Laptop");
    });

    it("should get all products", async () => {
        await Product.create({ name: "Phone", price: 499.99 });
        const res = await request(app).get("/api/products");
        expect(res.status).toBe(200);
        expect(res.body).toHaveLength(1);
        expect(res.body[0].name).toBe("Phone");
    });
});

Install dependencies: npm install --save-dev jest supertest mongodb-memory-server. Add "test": "jest" to package.json. Run: npm test.

Practical Applications

Testing ensures your e-commerce API is reliable, catching errors before users do. In this course, tests validate your API’s core functionality.

Tips for Success

To master testing:

  • Write Clear Tests: Focus on one behavior per test.
  • Use Mocks: Avoid real database calls in tests.
  • Run Frequently: Test after every change.
  • Check Coverage: Use jest --coverage.

Summary

Testing with Jest ensures your API’s reliability. By writing tests, you’ve strengthened your e-commerce app. In the next lesson, you’ll integrate your full-stack app. Keep practicing testing!
