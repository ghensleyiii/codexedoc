Project: Build a Single-Page Task Manager App with React and TypeScript

Building a Single-Page Task Manager App

You’ve completed Lessons 18–26 of CODEXEDOC’s Full-Stack Development Course, mastering JavaScript fundamentals, ES6+, DOM manipulation, asynchronous JavaScript, functional programming, TypeScript, React, state management, and testing. In this project, you’ll apply these skills to build a single-page task manager app using React and TypeScript, a practical portfolio piece showcasing modern front-end development.

Why This Project Matters

Task manager apps are common in real-world development, requiring skills like state management, UI design, and testing. This project simulates a professional task, preparing you for building interactive web apps and demonstrating your abilities to employers.

Project Overview

You’ll create a single-page app that allows users to add, edit, delete, and filter tasks. The app will use React for the UI, TypeScript for type safety, Context API for state management, and Jest for testing. Requirements include:

  • React Components: Build reusable components (e.g., TaskList, TaskForm).
  • TypeScript: Use interfaces for task data (e.g., interface Task).
  • Context API: Manage tasks globally without prop drilling.
  • Functional Programming: Use map, filter for task rendering.
  • Asynchronous JavaScript: Simulate API calls with async/await.
  • Testing: Write Jest tests for task addition and filtering.
  • Responsive Design: Style with Tailwind CSS or SCSS, ensuring mobile compatibility.

Sample Implementation

Here’s a simplified example of the task manager app structure:

<html lang="en">

    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Task Manager</title>
</head>
<body>
    <div id="root"></div>
    <script src="app.js"></script>
</body>
</html>

<!-- App.tsx -->
import React, { createContext, useContext, useState } from 'react';
import ReactDOM from 'react-dom';

interface Task {
    id: number;
    text: string;
    completed: boolean;
}

interface TaskContextType {
    tasks: Task[];
    addTask: (text: string) => void;
}

const TaskContext = createContext({ tasks: [], addTask: () => {} });

const App: React.FC = () => {
    const [tasks, setTasks] = useState([]);
    const addTask = (text: string) => {
        setTasks([...tasks, { id: tasks.length + 1, text, completed: false }]);
    };
    return (
        <TaskContext.Provider value={{ tasks, addTask }}>
            <TaskForm />
            <TaskList />
        </TaskContext.Provider>
    );
};

const TaskForm: React.FC = () => {
    const [text, setText] = useState('');
    const { addTask } = useContext(TaskContext);
    const handleSubmit = () => {
        if (text) {
            addTask(text);
            setText('');
        }
    };
    return (
        <div>
            <input
                type="text"
                value={text}
                onChange={(e) => setText(e.target.value)}
                placeholder="Add task"
            />
            <button onClick={handleSubmit}>Add</button>
        </div>
    );
};

const TaskList: React.FC = () => {
    const { tasks } = useContext(TaskContext);
    return (
        <ul>
            {tasks.map(task => (
                <li key={task.id}>{task.text}</li>
            ))}
        </ul>
    );
};

ReactDOM.render(<App />, document.getElementById('root'));

<!-- App.test.tsx -->
import { render, screen, fireEvent } from '@testing-library/react';
import App from './App';

describe('Task Manager', () => {
    test('adds a task', () => {
        render(<App />);
        const input = screen.getByPlaceholderText('Add task');
        const button = screen.getByText('Add');
        fireEvent.change(input, { target: { value: 'New Task' } });
        fireEvent.click(button);
        expect(screen.getByText('New Task')).toBeInTheDocument();
    });
});

This code sets up a basic task manager with React, TypeScript, Context API, and a Jest test. Add styling with Tailwind CSS or SCSS and async API calls for full functionality.

How to Approach the Project

Follow these steps:

  • Setup: Initialize a React app with TypeScript (npx create-react-app my-app --template typescript).
  • Components: Create TaskForm, TaskList, and TaskItem components.
  • State Management: Use Context API to manage tasks globally.
  • Functionality: Implement add, edit, delete, and filter (e.g., show completed tasks).
  • Async: Simulate API calls with a mock JSON server or async functions.
  • Testing: Write Jest tests for key features using @testing-library/react.
  • Styling: Use Tailwind CSS or SCSS for a responsive, polished UI.

Tips for Success

To excel in this project:

  • Plan Components: Sketch the UI before coding.
  • Use TypeScript: Define interfaces for tasks and context.
  • Test Early: Write Jest tests alongside development.
  • Debug with Dev Tools: Use React Developer Tools to inspect state.
  • Ensure Responsiveness: Test on mobile and desktop sizes.
