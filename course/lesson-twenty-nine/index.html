Authentication and Authorization

Mastering Authentication and Authorization

Welcome to Lesson 29! Authentication verifies user identity, while authorization determines access rights. In this lesson, you’ll learn to implement authentication with JSON Web Tokens (JWT) and authorization in Express.

What are Authentication and Authorization?

Authentication confirms “who” a user is (e.g., via login), and authorization checks “what” they can do (e.g., admin access). JWT is a standard for secure token-based authentication in 2025.

Key Concepts

Focus on these techniques:

  • JWT: Generate and verify tokens for user sessions.
  • Middleware: Protect routes with authentication checks.
  • Password Hashing: Secure passwords with bcrypt.
  • Role-Based Access: Restrict actions based on user roles.

Code Example: JWT Authentication

Here’s an Express server with JWT authentication:


const express = require('express');
const jwt = require('jsonwebtoken');
const bcrypt = require('bcrypt');
const app = express();

app.use(express.json());

const users = [];
const SECRET_KEY = 'your-secret-key';

app.post('/register', async (req, res) => {
    const { username, password } = req.body;
    const hashedPassword = await bcrypt.hash(password, 10);
    users.push({ username, password: hashedPassword });
    res.status(201).json({ message: 'User registered' });
});

app.post('/login', async (req, res) => {
    const { username, password } = req.body;
    const user = users.find(u => u.username === username);
    if (user && await bcrypt.compare(password, user.password)) {
        const token = jwt.sign({ username }, SECRET_KEY, { expiresIn: '1h' });
        res.json({ token });
    } else {
        res.status(401).json({ message: 'Invalid credentials' });
    }
});

app.listen(3000, () => {
    console.log('Server running on port 3000');
});

This server registers users, hashes passwords, and issues JWTs on login.

Practical Applications

Authentication secures APIs for apps like blogs. In your blog API project, you’ll implement JWT to protect post creation endpoints.

Tips for Success

To master authentication:

  • Secure Secrets: Store SECRET_KEY in environment variables.
  • Use Bcrypt: Hash passwords securely.
  • Validate Tokens: Check JWTs in middleware.
  • Test with Postman: Send tokens in Authorization headers.

Summary

Authentication and authorization secure your APIs. By mastering JWT and middleware, you’re ready for protected routes. Next, you’ll explore advanced API design with REST and GraphQL. Keep securing your APIs!
