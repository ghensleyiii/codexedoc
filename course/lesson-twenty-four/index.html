JavaScript Modules

Welcome to Lesson 24 of CODEXEDOC’s Full-Stack Development Course! In this lesson, you’ll learn JavaScript modules to organize and reuse code. By the end, you’ll be able to split a counter app into modules.

Introduction

Modules are like a shepherd’s toolbox, keeping tools organized and reusable. JavaScript modules split code into files, using import and export for modularity. This lesson teaches you to structure code for scalability.

Key Concepts

This lesson covers module essentials:

  • Export: Use export to share functions or variables from a module.
  • Import: Use import to access exported items in other files.
  • Module Scope: Variables are private by default, avoiding global pollution.

Code Example

Below is an example of a counter app using modules. Note: This assumes a local server (e.g., Live Server) to handle module imports.

<!DOCTYPE html>
<html lang="en">
<head>
    <title>Counter App</title>
</head>
<body>
    <h1>Count: <span id="count">0</span></h1>
    <button onclick="increment()">Increment</button>
    <script type="module">
        import { increment, getCount } from './counter.js';
        window.increment = increment;
        document.getElementById('count').textContent = getCount();
    </script>
</body>
</html>

<!-- counter.js -->
let count = 0;
export function increment() {
    count++;
    document.getElementById('count').textContent = count;
}
export function getCount() {
    return count;
}

Practical Applications

Modules are used in apps like VS Code or Notion to organize complex codebases. For example, a dashboard app uses modules to separate UI and data logic. These skills are vital for full-stack developers to maintain large projects.

Tips for Success

Here are tips to master modules:

  • Use a local server: Modules require HTTP for imports to work in browsers.
  • Keep modules focused: Each module should handle a single responsibility.
  • Name exports clearly: Use descriptive names for imported functions.

Shearing Tip

Modules shear away code chaos with organized files, just as a shepherd keeps tools in order. Import and export make your codebase clean and reusable.
