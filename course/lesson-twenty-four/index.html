Integrating Databases with Applications

Welcome to Lesson 24! You’ll learn to connect SQL databases to Node.js apps. By the end, you’ll build an API to retrieve user data.

Introduction

Database integration is like a shepherd linking wool to the flock’s records. This lesson covers using mysql2 or pg to execute SQL queries from Express routes.

Key Concepts

Focus areas include:

  • Database Drivers: Use mysql2 for MySQL or pg for PostgreSQL.
  • Queries in Node.js: Execute SQL from Express.
  • Security: Prevent SQL injection.

Code Example

Here’s an Express API retrieving user data.

const express = require('express');
const mysql = require('mysql2/promise');
const app = express();

app.use(express.json());

const db = mysql.createPool({
    host: 'localhost',
    user: 'root',
    password: 'your-password',
    database: 'app_db'
});

app.get('/users', async (req, res) => {
    try {
        const [rows] = await db.query('SELECT * FROM users');
        res.json(rows);
    } catch (error) {
        res.status(500).json({ error: 'Database error' });
    }
});

app.listen(3000, () => console.log('Server running at http://localhost:3000'));

Install: npm install express mysql2. Use parameterized queries to avoid SQL injection.

Practical Applications

Integration powers apps like LinkedIn, fetching user profiles. APIs serve data for dashboards or e-commerce. These skills are critical for full-stack development.

Tips for Success

Master integration with these tips:

  • Use connection pools for performance.
  • Secure credentials in environment variables.
  • Test queries in a database client first.
