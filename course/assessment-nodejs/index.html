Testing Your Node.js and Express Proficiency

You’ve completed Lessons 27–33, covering Node.js, Express, authentication, API design, security, testing, and deployment. This practical exam tests your ability to build a secure task management API with Express, a common backend feature, integrating all these skills.

Why This Assessment Matters

Task APIs are foundational for apps like to-do lists, requiring secure, tested endpoints. This exam simulates a real-world task, ensuring you can build production-ready backends, a key skill for full-stack developers.

Exam Overview

You’ll create a REST API for managing tasks, with endpoints for CRUD operations, JWT authentication, and Supertest tests. Use a mock database or MongoDB. Requirements include:

  • REST Endpoints: Create /api/tasks for CRUD operations.
  • Authentication: Protect endpoints with JWT.
  • Security: Use environment variables, helmet, and input validation.
  • Testing: Write Supertest tests for all endpoints.
  • Error Handling: Return consistent error responses.

Sample Task: Building the Task API

Task: Create a REST API with endpoints for creating, reading, updating, and deleting tasks, protected by JWT. Include a Supertest test for task creation.

Sample Solution:


const express = require('express');
const jwt = require('jsonwebtoken');
const helmet = require('helmet');
const dotenv = require('dotenv');
const app = express();

dotenv.config();
app.use(helmet());
app.use(express.json());

const tasks = [];
const SECRET_KEY = process.env.SECRET_KEY || 'your-secret-key';

const authMiddleware = (req, res, next) => {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ message: 'No token' });
    try {
        const decoded = jwt.verify(token, SECRET_KEY);
        req.user = decoded;
        next();
    } catch (error) {
        res.status(401).json({ message: 'Invalid token' });
    }
};

app.post('/api/tasks', authMiddleware, (req, res) => {
    const { title } = req.body;
    if (!title) return res.status(400).json({ message: 'Title required' });
    const task = { id: tasks.length + 1, title, user: req.user.username };
    tasks.push(task);
    res.status(201).json(task);
});

app.get('/api/tasks', authMiddleware, (req, res) => {
    const userTasks = tasks.filter(task => task.user === req.user.username);
    res.json(userTasks);
});

app.listen(3000, () => {
    console.log('Server running on port 3000');
});


const request = require('supertest');
const app = require('./server');

describe('Task API', () => {
    it('should create a task with valid token', async () => {
        const token = 'valid-jwt-token'; // Mock token
        const res = await request(app)
            .post('/api/tasks')
            .set('Authorization', `Bearer ${token}`)
            .send({ title: 'New Task' })
            .expect(201);
        expect(res.body.title).toBe('New Task');
    });
});

This API includes protected task endpoints and a Supertest test. Add full CRUD, user routes, and MongoDB for completion.
