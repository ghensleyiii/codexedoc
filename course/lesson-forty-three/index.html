NoSQL Databases with MongoDB

Mastering NoSQL with MongoDB

Welcome to Lesson 43! NoSQL databases like MongoDB offer flexible, schema-less storage for modern apps. In this lesson, you’ll learn MongoDB’s document model, queries, and use cases, complementing relational databases.

What is MongoDB?

MongoDB, launched in 2009, is a NoSQL database storing data as JSON-like documents. In 2025, it’s popular for Node.js and Python apps needing scalable, unstructured data storage.

Key Concepts

Focus on these features:

  • Collections: Groups of documents, like tables.
  • Documents: JSON-like data records.
  • Queries: Filter and aggregate data with MongoDB’s query language.
  • Indexes: Improve query performance.

Code Example: MongoDB Queries

Here’s a MongoDB query in the MongoDB shell:

// Insert documents into products collection
db.products.insertMany([
    { name: "Laptop", price: 999.99, stock: 10 },
    { name: "Phone", price: 499.99, stock: 20 }
]);

// Query products with stock > 15
db.products.find({ stock: { $gt: 15 } }).pretty();

// Update product price
db.products.updateOne(
    { name: "Laptop" },
    { $set: { price: 1099.99 } }
);

// Create index on name
db.products.createIndex({ name: 1 });

Run in the MongoDB shell to manage product data.

Practical Applications

MongoDB is ideal for flexible data like inventory logs. In your inventory management project, you’ll use MongoDB for transaction logs, complementing PostgreSQL.

Tips for Success

To master MongoDB:

  • Install MongoDB: Use MongoDB 7.0 (2025 standard).
  • Use MongoDB Compass: Visualize data and queries.
  • Index Fields: Index frequently queried fields.
  • Test Queries: Experiment in the MongoDB shell.

Summary

MongoDB offers flexible NoSQL storage. By mastering its document model, you’re ready for modern apps. Next, you’ll integrate databases with Node.js and Python. Keep exploring MongoDB!
